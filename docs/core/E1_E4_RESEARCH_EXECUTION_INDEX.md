# E1-E4 Research Execution Index

## Purpose

This is the working index for running SPHERE II entrepreneurship research step by step.

It answers:

- what lanes are fixed
- what can be researched first
- what should become public cases, validation guides, or founder playbooks

## Core rule

Do not collect startup ideas for their own sake.

SPHERE II is useful only when it turns opportunity noise into:

- clearer problem framing
- validation logic
- market or ecosystem evidence
- practical founder movement

## SET interpretation

Entrepreneurship can stand alone as `E`, but often appears as:

- `S+E` when research becomes venture opportunity
- `E+T` when a product needs technical implementation
- `S+E+T` when science, validation, and tools converge

The next useful task is to choose one E-route and build an evidence-backed `.md` output, not to keep adding abstract startup categories.
