## ⚔️ Quests

### 🗝️ Quest E4.1 — Weekly Update Template
**⏱ Time:** ~20 min | **Difficulty:** ⭐ Beginner

**Quest:** Create a repeatable template for founder updates.

```
WEEKLY UPDATE — Week [#]
├── ✅ Wins: What went well this week
├── 🚧 Blockers: What's stuck
├── 📊 Key Metric: [metric name] = [value] (↑/↓/→ from last week)
├── 🎯 Next Week's Focus: One priority
└── 🙏 One Ask: What do you need from [investors/advisors/team]?
```

**Rule:** Keep it under 200 words. Nobody reads long updates.

---

### 🗝️ Quest E4.2 — Public Case Structure
**⏱ Time:** ~30 min | **Difficulty:** ⭐⭐ Intermediate

**Quest:** Convert one founder story into a structured case under 500 words.

```
PUBLIC CASE
├── 📋 Background: Who, what, when
├── 🎯 Decision: What choice was made
├── 📈 Outcome: What happened (be honest)
└── 💡 Lesson: What would you do differently
```

### 🗝️ Quest E4.5 — Post-Mortem
**⏱ Time:** ~30 min | **Difficulty:** ⭐⭐⭐ Advanced

**Quest:** Write a post-mortem for a failed feature or product.

```
POST-MORTEM: [Feature/Product name]
├── 🎯 What we expected
├── ❌ What actually happened
├── 🔍 Why (root cause, not symptoms)
├── 💡 What we'd do differently
└── 📋 3 actionable changes for next time
```

**Reflection:** Can you be honest about failure without being paralyzed by it?
